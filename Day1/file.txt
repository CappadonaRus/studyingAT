
1. Есть программа, которая принимает с консоли три числа a, b, c и возвращает действительные корни уравнения a*x^2+b*x+c=0. 
Какие тест кейсы вы бы предложили для тестирования такой программы? Какие вопросы вы бы задали аналитику перед разработкой тест-кейсов?

- Проверить что программа запускается
- Написать метод который вычисляет дискриминант (D = b^2 - 4ac), передавать в метод входные значения и исходя из ответов(D>0, D=0, D<0) 
- Эти же числа передать в программу и сравнить результаты с ожидаемыми. (test case лежит в этой же папке как пример)
- Проверить что метод принимает не меньше и не больше 3х чисел. (пример этого test case лежит в этой же папке)
- Проверить корректность работы программы при вводе трех отрицательных / десятичных значений
- Проверить что значение а не равно 0
- Проверить отказ метода при передаче текстовых строк в каждый из предположительных чисел
- Проверить отказ метода при передаче различных спец символов.

Вопрос аналитику:
Есть ли ограничение на числа а, b, c
Какое ожидаемое поведение программы при получении отрицательного дискриминанта.

3. На вебсайте представлена форма вывода денег со счёта. 
Какие тест-кейсы вы бы предложили?
Какие вопросы вы бы задали аналитику перед разработкой тест-кейсов?

- Проверить что программа работает корректно при вводе правильных значений
- Проверить что деньги поступают на счет
- Проверка что баланс уменьшается
- Проверка что у клиента есть деньги на счету
- Проверить что программа меняет на правильную валюту

- Проверить работу с отрицательным значением

- Проверить что при нажатии "Cancel" программа корректно отменяет операцию

Входит ли в диапазон значений цифра 100, что должно происходить если пользователь ввёл значение 100.
